# my-dev-containers

Here is a clean, scannable summary of the exact configurations and steps required on your **Ubuntu host** to make this entire local containerized development sandbox run smoothly.

You can save this as a reference or a checklist for the future.

---

## Ubuntu Host Setup Summary

### 1. Engine Installation & Configuration

Instead of using Podman Desktop on Windows, the environment relies directly on the native Linux CLI engine inside your Ubuntu WSL2 distribution.

* **Install Podman:** Installed the core container runtime directly via `apt` to run rootless containers.
* **Network Driver (`pasta`):** Utilizes Podman 5's native `pasta` network stack for rootless operations, eliminating the need for root permissions while automatically handling host-to-container DNS resolution.

### 2. Enabling Systemd & The User Socket

Because you are running rootless containers, Podman requires a persistent user-level systemd service to manage container lifecycles and expose the engine API socket to your development environment.

* **Enable Systemd in WSL:** Ensure systemd is active on the Ubuntu host by adding this block to `/etc/wsl.conf`:
```ini
[boot]
systemd=true

```


* **Enable the Podman User Service:** Run the systemd commands to start and enable the rootless socket manager on boot:
```bash
systemctl --user enable --now podman.socket

```


* **Verify User ID Alignment:** Confirmed that your primary Ubuntu user maps to UID `1000`, ensuring the active socket is hosted at the exact path:
`/run/user/1000/podman/podman.sock`

### 3. Host Network Listening Rules

To prevent network isolation issues when your microservices need to cross boundaries (e.g., containers talking back to services running directly on the host, or a gateway like Tyk redirecting to external processes):

* **Interface Binding (`0.0.0.0`):** Any host-level or Windows-level service interacting with this cluster must be configured to listen on `0.0.0.0` (all interfaces) rather than exclusively locking to `127.0.0.1`.
* **DNS Gateway Integration:** Leverages the native `host.containers.internal` loopback alias auto-generated by the Ubuntu Podman engine inside container `/etc/hosts` files to allow seamless container-to-host communication.
